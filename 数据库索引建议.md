# 数据库索引建议

你看到的“可能缺乏高效的索引支持”提示，通常来自云开发数据库对 **`where(...) + orderBy(...)`** 组合查询的自动检测。

在生产环境，如果某些查询会频繁触发，建议为对应集合建立 **复合索引**（Composite Index），以避免全表扫描、慢查询和告警。

## pressure_records（压力表记录）

**A. 企业压力表库列表（按创建时间倒序）**
- 触发位置：`pages/archive/archive.js` 的 `loadRecords()`
- 查询形态：`where({ enterpriseName })` + `orderBy(createTime desc)`
- 建议索引：`enterpriseName (ASC) + createTime (DESC)`

**A2. 企业维度统计计数（按企业 + openid）**
- 触发场景：企业侧/云函数统计时使用 `enterpriseName + _openid` 双条件 `count()`
- 查询形态：`where({ enterpriseName, _openid }).count()`
- 建议索引：`enterpriseName (ASC) + _openid (ASC)`

**B. 设备关联计数**
- 触发位置：`services/device-service.js` 的 `updateRecordCount()`
- 查询形态：`where({ deviceId })` + `count()`
- 建议索引：`deviceId (ASC)`

**C. 到期提醒（企业维度，按到期日升序）**
- 触发位置：`cloudfunctions/expiryReminder/index.js`
- 查询形态：
  - `where({ enterpriseName, status, expiryDate < now })` + `orderBy(expiryDate asc)`
  - `where({ enterpriseName, status, expiryDate in [now, threshold] })` + `orderBy(expiryDate asc)`
- 建议索引：`enterpriseName (ASC) + status (ASC) + expiryDate (ASC)`

**D. 到期提醒（管理员维度，按到期日升序，可选辖区）**
- 触发位置：`cloudfunctions/expiryReminder/index.js`
- 查询形态：
  - `where({ district?, status, expiryDate ... })` + `orderBy(expiryDate asc)`
- 建议索引（按实际使用选择其一或两者都建）：
  - `district (ASC) + status (ASC) + expiryDate (ASC)`
  - `status (ASC) + expiryDate (ASC)`（不按辖区过滤时）

## devices（设备表）

**A. 企业设备列表（按创建时间倒序）**
- 触发位置：`services/device-service.js` 的 `loadDevices()`
- 查询形态：`where({ enterpriseName })` + `orderBy(createTime desc)`
- 建议索引：`enterpriseName (ASC) + createTime (DESC)`

**B. 管理员按辖区查设备（按创建时间倒序）**
- 触发位置：`services/device-service.js` 的 `loadDevices()`（管理员模式）
- 查询形态：`where({ district })` + `orderBy(createTime desc)`
- 建议索引：`district (ASC) + createTime (DESC)`

## enterprises（企业表）

**企业汇总里按 companyName in(...) 查询**
- 触发位置：`cloudfunctions/expiryReminder/index.js` 的 `getExpiringSummary()`
- 查询形态：`where({ companyName: in([...]) })`
- 一般单字段索引即可：`companyName (ASC)`

## 如何创建索引（云开发控制台）

1. 打开 **云开发控制台 → 数据库**
2. 选择集合（如 `pressure_records`）
3. 打开 **索引管理 / Index**
4. 新建索引：按上方建议添加字段与排序（ASC/DESC）
5. 保存并等待索引构建完成

> 说明：包含 `RegExp`/`$or` 的模糊搜索通常无法完全依赖索引提升性能；建议将“列表页常用查询”先用索引覆盖，搜索功能可按需优化。
